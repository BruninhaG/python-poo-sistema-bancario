# Sistema Bancário com Programação Orientada a Objetos (POO) em Python

## 📌 Descrição
Este projeto faz parte da evolução de um sistema bancário simplificado. O foco principal foi aplicar os pilares da **POO** para criar um código mais organizado, reutilizável e fácil de manter. Saímos de uma estrutura de dados simples para um modelo de classes que simula o mundo real.

## 🏗️ Modelagem UML
O sistema foi estruturado seguindo o modelo de classes que inclui:
* **Cliente e Pessoa Física:** Aplicação de herança.
* **Conta, Conta Corrente e Histórico:** Relacionamentos entre objetos.
* **Transações (Depósito e Saque):** Implementação de interfaces para padronizar operações.

## 🛠️ Tecnologias e Conceitos Aplicados
* **Linguagem:** Python
* **Pilares da POO:**
    - **Herança:** Criação de especializações (ex: Pessoa Física).
    - **Encapsulamento:** Proteção e organização dos dados das contas.
    - **Polimorfismo:** Diferentes formas de realizar transações.
    - **Abstração:** Focar no que é essencial para o sistema bancário.

## 🚀 Como funciona
O sistema permite cadastrar usuários (clientes), abrir contas bancárias vinculadas a esses usuários e realizar operações de depósito, saque e visualização de extrato, onde cada operação é registrada como um objeto no histórico da conta.
